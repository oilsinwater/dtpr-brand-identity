01 · Evolution

How the visual identity got here

Six years of design work, four major surfaces, two visual systems in tension. This page is the timeline — what was made when, what each version contributed to the brand, and what the trajectory tells us about where it should go next.

Timeline

2020 · Sidewalk Labs / Alphabet era
vision.dtpr.io
A speculative co-design prototype developed during DTPR's time inside Sidewalk Labs. Long horizontal-scrollytelling page, seven "How might we..." questions, each opening into a scenario where a person interacts with a place where DTPR is deployed. The hex mark, the data chain composition, the 147-element taxonomy UI, and the teal "identifiability" color all originate here.
  • Mark: canonical hexagon — three SVG variants (blue, black, outline)
  • Color: Bootstrap-extracted palette; navy primary, teal identifiability, pale teal secondary
  • Type: system stack, no custom body font
  • Layout: horizontal-scroll scrollytelling with floating UI mockups
  • Footer: minimal — no stewardship signature

2024–2026 · Public standard site
dtpr.io
The current public-facing site. Problem → Proposal → Mission narrative. Built in Nuxt + Tailwind. The most-trafficked of the surfaces (2,300+ unique visitors/year) but with a 35s / 49% scroll-depth bounce on the homepage and a critical QR-scan dead-end on /taxonomy/device. The Corporate Memphis flat-illustration hero is the Sidewalk Labs / Alphabet holdover that Phase 2 deprioritized for new work.
  • Type: Red Hat Text, Google Fonts at 400/500/700
  • Color: navy primary, teal accent; Bootstrap yellow/red/orange available but unused as brand colors
  • Mark: small dark hexagon in header (Nuxt-rendered)
  • Hero: Corporate Memphis cityscape illustration
  • Footer: CC BY 4.0 + Helpful Places stewardship

May 19, 2026 · Assembly Day convening
DTPR Assembly Day deck
A 31-page internal brand + strategy presentation created for the post-grant POSE convening. Most material evidence: the canonical Corporate Memphis cityscape as the *most complete* in the system (a transit/bike/storefront scene with recycling station, lampposts, restaurant — used in the "DTPR for AI Community Conversation Kit" cover). Also surfaces an additional cobalt-blue accent color (#2D2BFF) used for "Communication" callout cards, and teal vertical column dividers in the journey-mapping layouts. This is a brand layer the web surfaces do not currently show.
  • Color additions: cobalt #2D2BFF for "Communication" tags; amber for "Capacity Building"
  • Layout grammar: 3-column journey maps with vertical teal rules
  • Hero: the most-refined Corporate Memphis cityscape in the system
  • Format: presentation deck — not a live site

2026 · MakeShift event
makeshift2026.dtpr.io
The 2026 MakeShift event site (Accountable Tech by Design). The newest expression of the visual identity — and the cleanest. Built for a one-week convening in NYC. Single CSS file, no framework, no build pipeline. The reference for "what DTPR could look like if we built it from scratch today."
  • Type: JetBrains Mono (display, UI) + Source Sans (body), both self-hosted
  • Color: explicit token system — --navy, --gov-blue, --royal-blue, --pale-blue, --lime, --ink
  • Mark: layered hexagonal "city map" with axis labels — event-specific
  • Background: subtle blue grid pattern (rgba(65,105,225,0.11))
  • Layout: explicit color-token sections; sticky nav with backdrop blur; parallax

OBSERVED Live at makeshift2026.dtpr.io

2024–2026 · Documentation
docs.dtpr.io
The standard's documentation site. Built on Mintlify. Light mode by default — white background, dark text, teal link accents (the same #0f5153 identifiability color from the Bootstrap palette, not a Mintlify default). Different layout from the marketing sites — three-column with left sidebar, main content, right rail — but the same hex mark in the header and the same brand palette.
  • Type: Mintlify's system font stack
  • Color: white background, near-black text, teal #0f5153 link accent
  • Layout: left-sidebar nav with sectioned content; "On this page" right rail
  • Mark: the canonical DTPR hex mark in the top header (dark-on-white)

What the trajectory tells us

Pattern: each site picked up one layer and dropped the others.

vision.dtpr.io brought the hex mark and the taxonomy. dtpr.io brought the stewardship footer and the Public Sector trust signals. Makeshift brought explicit color tokens and a JetBrains-Mono-as-UI discipline. No single site carries the full system. The brand book has to combine them.

Pattern: visual systems multiplied, never consolidated.

Corporate Memphis (dtpr.io, Assembly Day) coexists with hex-based scenarios (vision) and the layered hex mark (Makeshift). The brand book has to pick — or to name which goes where.

Pattern: the most disciplined design is the one with the fewest constraints.

Makeshift is a single-CSS-file event site. It is the most carefully designed surface in the ecosystem. The lesson: constraint produces coherence.

Pattern: navy is the only constant.

#002684 appears in all four light-mode sites. It is the brand. Everything else is negotiable.
