What the brand book has to decide OPEN

One stack or a typographic grammar?

The three sites each loaded what worked for their moment. The Phase 3 brand sprint needs to commit to one — or to name the grammar (e.g. "humanist sans for body, mono for UI labels, system fallback") that lets each context pick.

Candidate: The MakeShift three-role stack (below) — Source Sans Variable for body, JetBrains Mono for UI labels, Playfair Display italic for editorial headlines. PROPOSED MEMO Sourced from brand foundation memo v2, §5.

MakeShift candidate type system PROPOSED

The MakeShift 2026 assets establish a three-role type system across all formats. This is the strongest candidate for Phase 3 — it's already battle-tested in print, signage, and web contexts. All three fonts are open-source.

Accessibility notes OBSERVED

  • Body line-height is consistently 1.6–1.75 across sites — good for sustained reading.
  • dtpr.io uses clamp() for responsive display sizing, which scales smoothly across viewports.
  • Body weight 400 + line-height 1.6 is the accessibility floor. Anything below this fails WCAG AA reading-grade for low-vision readers.
  • No body text on the three sites uses fonts below 0.95rem. Don't go lower.
