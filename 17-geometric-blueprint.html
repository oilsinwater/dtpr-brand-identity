DTPR Hexagon: Geometric Blueprint

The DTPR Hexagon Unit is the foundational geometry of the entire visual system — the atom from which signage panels, icon sets, grid structures, and digital layouts are derived. This document specifies the mathematical constraints that ensure consistency across all three sprint tracks.

Master Unit — Flat-Top Hexagon HEX-UNIT-01 · Scale-Agnostic
/* DTPR Hex Unit — Scale-Agnostic Formula */ const R = unit_radius; // base unit — set per context width = 2 × R; // flat-to-flat diameter height = R × √3; // ≈ R × 1.732 (point-to-point) stroke = R × 0.02; // border stroke weight corner = R × 0.04; // vertex softening radius inset = R × 0.16; // content safe area margin gap = R × 0.08; // inter-hex spacing in grid

Blueprint Rationale

  1. Scale-agnostic formula: All values are expressed as R multiples. This means a single spec governs everything from a 16px favicon to a 500mm outdoor sign — the ratio relationships stay constant.
  2. Flat-top orientation: DTPR hexes are flat-top (not pointy-top) following the ISO 3864 sign shape convention and the existing Makeshift visual language. This keeps horizontal icon rows consistent.
  3. Stroke ≠ border: The stroke is part of the form, not a container. At small sizes (<16px), use fill-only with no stroke. At large physical sizes (>100mm), stroke weight should not exceed 2mm regardless of R×0.02 calculation.
  4. Grid inheritance: Sprint Recipe B (Maker Track) uses a fluid grid overlay from this tiling spec. Recipe A (Regulatory) locks the tiling to a fixed ISO modular grid. Recipe C (Commons) uses single-column with no tiling.
