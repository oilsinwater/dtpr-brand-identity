5 Core Application Assets

In Lorenz's CAA method — Components → Assets → Application — the final stage applies assembled assets (Symbol, Line, Frame, Label) onto real formats and mediums. These five assets are DTPR's highest-impact Application deliverables: the physical and digital touchpoints where the system reaches its audiences and where design decisions matter most. Each is derived from a Sprint Recipe (p.16) and constrained by the Geometric Blueprint (p.17).

A note on "CAA": Throughout this book, CAA is Lorenz's Components / Assets / Application workflow. The CAA-01…CAA-05 identifiers below name the Application-stage outputs — DTPR's core communication & action touchpoints. The acronym refers to the method, not a separate "Communication & Action Assets" coinage.

Assembled Panel — CAA-01 at Full Specification

Panel Assembly Notes

  • Top bar: DTPR wordmark + full name + domain — always navy, always present
  • Hex grid: 1–5 icons depending on technology count. Grid scales to fill width.
  • QR code: always right-aligned, minimum 48×48mm at 0.5m reading distance
  • Footer: operator identity + purpose statement + data retention + stewardship attribution
  • Chartreuse accent strip: 5mm, bottom edge — signal color, always present, cross-track consistent
  • Total min. panel size: 120×80mm (2 hex icons). Standard: 210×140mm (5 hex icons)

CAA Design Rationale

  1. Five not twelve: The consolidation plan identifies 24 high-priority items across 84 assets. These 5 CAA assets address the top 4 Q1 matrix gaps (QR onboarding, deployment playbook, developer portal, feedback portal) plus the physical sign — the universal anchor across all tracks.
  2. Recipe inheritance: Each CAA inherits its full design spec from a Sprint Recipe. CAA-01 and CAA-04 use Recipe A (Regulatory). CAA-02 and the feedback portal extension use Recipe C (Commons). CAA-03 and CAA-05 use Recipe B (Maker). This prevents visual fragmentation without forcing one system on all contexts.
  3. Chartreuse as universal thread: The accent strip on CAA-01 appears as the CTA button on CAA-02 and the selected state on CAA-03. It is the one visual element that crosses all tracks — the "signal" in the system.
