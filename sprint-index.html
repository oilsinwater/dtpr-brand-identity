How to read this system prep

Nine pages detailing DTPR's generative design engine. We start with three-dimensional morphological boxes (Form, Process, Semantic), combine them into target recipes, blueprint the hexagon geometry, construct core assets, align with ISO warning rules, and specify responsive breakpoints.

The Grounding Persona Loop

Our design parameters are mapped directly against four primary stakeholder groups. By matching design columns to user-centered motivations, we ensure the visual systems address real-world needs:

  • Pragmatic Deployers (CIOs, City Leaders): Require strict safety compliance, risk checklists, and procurement-ready layouts.
  • Vigilant Advocates (Civic Tech, Non-Profits): Need decentralized trust systems, clear data hierarchies, and collaborative feedback loops.
  • Impacted Citizens (Scanning Public): Require maximum simplicity, plain language, and neutral static wayfinding labels.
  • Technical Evaluators (Developers): Need modular API specifications, monospaced code blocks, and copy-pasteable assets.

System Alignment

Unified Parameters

By organizing design decisions into parameter matrices, we ensure that new DTPR components inherit verified grid, color, and warning rules systematically.

ISO Safety Checked

Our parameters actively map safety standards, prohibiting color collisions (like amber warnings in safety zones) and enforcing co-placement distance minimums.

Physical to Digital

Responsive rules are tied directly to viewing zones (10m, 2m, 0.5m) and digital viewports (320px, 428px, 768px, 1280px) to guarantee seamless scanning loops.
